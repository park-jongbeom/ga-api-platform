# 📜 API Tool & Development Policy (Backend)

## 1. AI 도구 개발 지침
1. 한국어로 답해줘 (모든 응답, 문서화, 주석)
2. 외부 라이브러리(Spring AI, LangChain4j 등)나 특정 알고리즘 사용 시 참고자료 출처(URL) 명시
3. '중대한 변화' 시 브랜치 생성 및 History 폴더 기록 (신기능 구현, DB 스키마 변경, API 규격 전면 수정)
4. 모든 기능 구현 시 JUnit5, MockK 등을 활용한 **테스트 코드** 동반 구현 필수

## 2. 커밋 메시지 규칙
커밋 메시지는 반드시 아래 형식을 포함하여 충실하게 작성한다.
- **내가 요구한 내용**: (유저의 요청 프롬프트 요약)
- **네가 답한 내용**: (AI가 제안한 로직 및 설계 방향)
- **수정한 내용 요약**: (변경된 클래스, 메서드, DB 마이그레이션 파일 등 상세)

## 3. Problem 1-Pager (작업 시작 전 필수 작성)
복잡한 비즈니스 로직(AI 상담 에이전트 등) 시작 전, 다음 항목을 포함한 1-Pager를 작성한다.
- 배경(Background): 변경 맥락과 동기
- 문제(Problem): 해결하려는 이슈 (예: RAG 검색 정확도 개선)
- 목표(Goal): 성공의 기준 (예: 테스트 커버리지 80%, API 응답 속도 등)
- 비목표(Non-goals): 범위 밖의 사항
- 제약(Constraints): 보안 수칙 및 기술 스택 제약
